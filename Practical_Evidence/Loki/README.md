Loki practical evidence files:

Loki was used as the centralized log aggregation system during the DDoS investigation.

It collected logs generated by Zeek and allowed real-time monitoring of suspicious network activity through Grafana.

Unlike traditional logging systems, Loki stores logs efficiently using label-based indexing, which made searching faster and easier during forensic analysis.

Promtail was used to forward system and Zeek logs to Loki, while Grafana was used to visualize and query those logs using LogQL.

The query {job="zeek"} helped identify suspicious communication between the source and destination systems, especially traffic involving port 5001.

Loki logs showed unusual events such as active_connection_reuse, truncated_tcp_payload, and connection abnormalities that supported attack timeline reconstruction.

This helped validate attack initiation, peak activity, and recovery phases and improved the accuracy of the final forensic investigation.
